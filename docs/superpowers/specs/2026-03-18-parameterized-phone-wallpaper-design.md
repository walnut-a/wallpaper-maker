# 参数化手机壁纸生成工具设计文档

## 1. 背景

目标是在当前仓库中实现一个参数化生成手机壁纸的纯前端工具，严格遵守 [PRINCIPLES.md](/Users/zhaolixing/Documents/GitHub/wallpaper-maker/PRINCIPLES.md) 中定义的边界：

- 单文件优先，尽量收敛为一个可直接托管的 `HTML` 文件
- 静态托管环境完整可用，可直接部署到 GitHub Pages
- 页面打开即用，不依赖后端、本地服务或构建链
- 结果优先，让壁纸预览始终成为界面的视觉中心

这个项目第一版不追求做成一个复杂设计平台，而是做成一个轻量、打开就能生成和导出壁纸的创作工具。

## 2. 目标与非目标

### 2.1 目标

- 提供一个单文件、纯浏览器运行的手机壁纸生成工具
- 默认打开页面时直接展示一张可用的成品壁纸
- 支持四种几何类风格预设
- 支持自定义宽高，并附带常用手机尺寸预设
- 支持常用参数与高级参数的双层控制
- 支持实时预览和本地导出 `PNG`

### 2.2 非目标

- 不做账号、云同步、作品图库或分享社区
- 不做依赖服务端的生成流程
- 不做多页面应用或复杂工程化构建
- 不在第一版支持文案海报、照片混合、图层导入等非几何主线功能
- 不在第一版引入第三方大型图形库

## 3. 用户体验原则

### 3.1 打开即看到结果

页面初始状态不应为空。用户首次打开页面时，应直接看到一张已经生成好的手机壁纸，以及一组合理的默认参数。默认状态本身就是产品态度的一部分，不能像调试页。

### 3.2 控制区服务结果，不抢结果

桌面端采用左侧控制面板、右侧大预览的布局。移动端改为上方预览、下方参数。预览区是视觉中心，控制区保持克制，避免出现“工具界面比作品更抢眼”的问题。

### 3.3 快速闭环

首页只保留最短主链路：

1. 打开页面
2. 看到成品壁纸
3. 调整参数或切换风格
4. 导出 `PNG`

尽量不在首屏堆砌冗长说明，不要求用户先理解系统概念才开始使用。

## 4. 产品范围

第一版聚焦为“参数化几何壁纸生成器”，不扩展到文字海报或图像拼贴。风格上内置四种预设，但它们共享同一套底层绘制能力和统一参数模型。

### 4.1 预设列表

1. 极简秩序感
2. 大色块构成
3. 科技纹理感
4. 有机生成感

### 4.2 核心动作

- 切换风格预设
- 调整画布宽高或选择常用尺寸预设
- 修改常用参数
- 展开高级参数进行细调
- 重新生成
- 随机参数
- 重置当前预设
- 导出 `PNG`

## 5. 信息架构

### 5.1 桌面端布局

- 左侧：控制面板
- 右侧：大预览区

控制面板从上到下建议按以下顺序组织：

1. 预设切换
2. 画布尺寸
3. 常用参数
4. 快捷动作
5. 高级参数折叠区

这样顺序符合真实使用流程：先定风格，再定尺寸，再微调，再导出。

### 5.2 移动端布局

- 上方：预览区
- 下方：控制面板

移动端不直接压缩桌面端结构，而是按“先看结果、再动参数”的使用顺序重新组织。控制面板使用分组和折叠，避免出现超长表单。

## 6. 参数体系设计

### 6.1 总体策略

采用“统一参数模型 + 预设默认值 + 参数显隐规则”的结构。

统一参数模型解决两件事：

- 四种风格共用同一套底层状态和导出流程
- 新增风格时优先通过配置扩展，而不是重新发明一套 UI 与绘制逻辑

每个预设配置至少包含：

- 名称
- 默认参数值
- 暴露给默认层的参数列表
- 暴露给高级层的参数列表
- 参数范围与步进
- 绘制步骤与图层权重

### 6.2 常用参数层

默认展示的常用参数建议包括：

- 风格预设
- 宽度
- 高度
- 尺寸预设
- 调色板
- 密度
- 复杂度
- 随机种子

这层参数应该让用户在不理解全部系统的情况下，也能快速做出明显变化。

### 6.3 高级参数层

高级参数通过折叠区展开，面向想深度调节的用户。建议包括：

- 图形类型权重，如线条、圆、矩形、弧线、放射元素占比
- 网格细分与对齐方式
- 留白边距与安全区
- 渐变方向与强度
- 纹理项，如颗粒、扫描线、噪点、模糊
- 图层透明度与混合方式
- 随机扰动幅度
- 对称与重复规则

高级参数变化同样应实时更新预览，不需要单独点击“应用”。

## 7. 四种预设定义

### 7.1 极简秩序感

视觉特点：

- 细线
- 网格
- 大留白
- 低饱和渐变

默认常用参数建议：

- 线密度
- 留白比例
- 色彩对比
- 网格尺度
- 对齐强度

### 7.2 大色块构成

视觉特点：

- 矩形、圆形、弧面
- 强块面关系
- 重叠和遮挡
- 对比更强

默认常用参数建议：

- 图形数量
- 块面大小
- 重叠程度
- 主副色比例
- 边角圆润度

### 7.3 科技纹理感

视觉特点：

- 放射线
- 扫描线
- 局部高亮
- 颗粒与密度变化

默认常用参数建议：

- 纹理强度
- 线束密度
- 中心偏移
- 亮部范围
- 颗粒粗细

### 7.4 有机生成感

视觉特点：

- 波纹
- 曲线
- 流动层
- 轻微随机扰动

默认常用参数建议：

- 波动幅度
- 流线数量
- 平滑程度
- 层叠透明度
- 扰动强度

## 8. 技术方案

### 8.1 引擎选择

第一版使用 `Canvas` 作为唯一绘制引擎。

选择理由：

- 更适合高分辨率手机壁纸位图输出
- 预览与 `PNG` 导出链路简单
- 纯前端单文件更容易控制
- 对噪点、纹理、叠层等视觉效果更友好

不采用 `SVG` 或混合方案，原因是第一版核心目标是快速建立稳定的生成与导出闭环，而不是追求矢量编辑能力。

### 8.2 单文件内部结构

虽然最终交付为单文件 `HTML`，内部逻辑仍按职责分层：

1. 状态层  
保存当前预设、画布尺寸、参数值、随机种子与导出配置。

2. 预设定义层  
存放四种预设的默认值、参数范围、绘制规则与显隐配置。

3. 绘制引擎层  
统一使用 `Canvas` 根据当前状态执行绘制。

4. 参数面板层  
基于配置渲染控件，不用写死一大批条件判断。

5. 导出层  
通过离屏 `Canvas` 按目标分辨率重新绘制并导出。

6. 工具层  
提供种子随机、颜色插值、尺寸映射、噪点生成等纯函数。

### 8.3 绘制管线

建议使用统一图层顺序：

1. 背景层
2. 基础几何层
3. 强调结构层
4. 纹理层
5. 可选收尾层，如局部叠光或暗角

四种预设共享这套管线，但每层的启用情况、权重与算法不同。

### 8.4 导出策略

预览与导出必须共用同一套绘制逻辑，只允许尺寸不同，不允许逻辑分叉。导出时使用离屏 `Canvas` 重绘目标尺寸，再输出 `PNG`。

这样可以避免“页面预览”和“导出成品”表现不一致。

## 9. 默认体验

### 9.1 默认预设

建议将“极简秩序感”作为默认风格，因为它最稳，也最适合体现页面初始质感。

### 9.2 初始状态

打开页面时直接生成一张完成度高的成品壁纸，且所有默认参数都处在合理范围内。首屏不能出现空白画布、未初始化参数或测试用占位内容。

### 9.3 随机机制

保留两类随机行为：

- 重新生成：沿用当前核心参数，仅基于种子刷新画面
- 随机参数：为当前预设生成一组新的可用参数组合

随机结果应有边界控制，避免直接生成明显失衡或不可用的图像。

## 10. 异常与降级策略

### 10.1 输入异常

- 宽高非法时，就地提示并恢复到最近一次有效值
- 参数超出范围时自动钳制
- 非法状态不得导致整张画布空白

### 10.2 性能降级

当某些纹理或图层在高分辨率下成本过高时，优先保证预览与导出仍然可用，可对纹理密度、采样精度或局部效果进行降级。

### 10.3 导出失败反馈

导出失败时需要明确反馈失败结果，不能静默无响应。提示文案应让用户知道是浏览器限制、内存问题，还是当前尺寸过大。

## 11. 验证边界

第一版至少验证以下内容：

- 四种预设都能稳定生成，不出现空白图与严重错位
- 常用尺寸预设和自定义尺寸都能正常预览与导出
- 参数改动会实时触发重绘
- 桌面端和移动端布局符合真实使用顺序
- 纯静态环境打开即可用，不依赖后端

如果有额外时间，再补充：

- 高频参数拖动时的性能表现
- 高分辨率导出时的内存占用表现
- 各风格默认状态的视觉完成度

## 12. 实施建议

实现时优先按以下顺序推进：

1. 搭建单文件页面骨架与基础布局
2. 建立统一状态与参数模型
3. 打通 `Canvas` 预览与 `PNG` 导出
4. 先实现一个默认预设，验证参数面板与导出闭环
5. 再扩展剩余三个预设
6. 最后补高级参数、降级策略和细节打磨

这样可以尽快建立可运行主链路，避免前期把时间耗在大量风格细节上。

## 13. 结论

这个项目第一版的合理形态是：

一个单文件、纯前端、静态托管可用、打开即用的参数化手机壁纸生成器。它以 `Canvas` 为唯一绘制引擎，提供四种几何风格预设，采用“常用参数 + 高级参数”的双层控制结构，支持自定义尺寸和常用尺寸预设，并可在浏览器本地实时预览与导出 `PNG`。

这个方向与当前仓库的原则文档完全一致，既保住了轻量工具的边界，也给后续扩展预设和参数留出了清晰空间。
