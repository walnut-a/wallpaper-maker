# 材质主体融场化设计文档

## 1. 背景

上一轮已经把 `长虹玻璃柱面` 和 `镜面切片` 从“纯效果堆叠”改成了“先有主形体，再补材质提示”的方向，但实际出图仍然存在一个更深的问题：

- 它们还是像被贴到画面上的对象
- 没有真正和整张壁纸共享同一个视觉场

从当前成图来看：

- `长虹玻璃柱面` 像一根居中的独立物体，轮廓太完整
- `镜面切片` 像几片插进去的硬片，边界太独立、太僵

这次迭代不再继续修“对象渲染”，而是把两者分别改成更适合壁纸的“融场”方案。

## 2. 目标与非目标

### 2.1 目标

- 把 `长虹玻璃柱面` 改成全画面介质场，而不是单个柱体对象
- 把 `镜面切片` 改成从背景里浮出的半融化切面，而不是独立硬片
- 让这两个主体更能融入背景，并保住抽象壁纸本身的完成度
- 保持现有模块系统、单文件结构、Canvas 2D 实现和导出逻辑不变

### 2.2 非目标

- 这次不新增新的主体模块
- 不改现有 UI 结构
- 不改背景模块和纹理模块的注册方式
- 不追求写实物理渲染
- 不引入 WebGL、Shader 或外部依赖

## 3. 问题判断

### 3.1 长虹玻璃的问题

当前版本的失败点不只是“像不像玻璃”，而是它把玻璃当成了一个被放进画面里的对象：

- 完整闭合的轮廓太强
- 居中独立主体太明显
- 玻璃感只集中在那一根柱体内部

结果是：

- 用户先看到“一个对象”
- 再看到“这个对象像玻璃”

而不是先看到“整张画面都被同一种介质统治”

### 3.2 镜面切片的问题

当前版本已经有了切面提示，但它们彼此之间和背景之间的关系还是太生硬：

- 每片切面都像独立道具
- 面与面之间共享的流向不够
- 面和背景没有共同的明暗场

结果是：

- 用户先看到几片硬片
- 再勉强把它们理解成镜面

而不是看到背景里有几处切面正在浮出来

## 4. 方案比较

### 方案 A：继续修当前对象渲染

保留闭合柱体和完整切片，只微调边缘、模糊和高光。

优点：

- 改动小
- 风险低

缺点：

- 无法解决“贴在画面上”的结构问题
- 只会继续在对象表面堆效果

### 方案 B：两者都改成全画面场

`长虹玻璃` 和 `镜面切片` 全都弱化成整张背景结构。

优点：

- 更容易融入整张壁纸

缺点：

- `镜面切片` 会丢掉主体辨识度
- 两个模式会变得过于接近

### 方案 C：分别处理

- `长虹玻璃` 改成全画面介质场
- `镜面切片` 保留主体，但让边界半融化、与背景共享同一场

优点：

- 最符合当前实际问题
- 两者差异更清楚
- 更适合壁纸这种“背景优先、主体克制”的介质

缺点：

- 需要分别重写两套内部逻辑

### 推荐

采用 `方案 C`。

## 5. 长虹玻璃：全画面介质场

### 5.1 目标效果

用户第一眼看到的应该是：

- 整张画面都经过纵向介质
- 不同区域的玻璃感强弱不同
- 局部会有更强的折射和亮度压缩

而不是一根单独存在的玻璃柱体。

### 5.2 核心设计

`长虹玻璃` 从“对象”改成“场”之后，主体不再依赖完整轮廓，而依赖三层关系：

1. 全画面的竖向棱线场  
   线宽和间距有轻微呼吸感，不是机械重复。

2. 全画面的折射雾场  
   背景的大色团、亮区、暗区都会被纵向介质轻微压缩、拖拽和重新分布。

3. 局部增强区  
   只在一段或两段局部区域增强玻璃感，像厚度、密度或亮度更高，但不出现完整闭合形体。

### 5.3 实现顺序

1. 先生成背景本身的大色域和雾场
2. 再叠一层全画面的长虹介质场
3. 最后用局部增强区去制造重心

### 5.4 参数调整

保留或新增：

- `棱线密度`
- `棱线呼吸`
- `全局折射`
- `局部增强`
- `雾场融合`

移除：

- `柱面数量`
- `柱体宽度`

## 6. 镜面切片：半融化切面

### 6.1 目标效果

用户第一眼看到的应该是：

- 背景里浮出几处切面
- 切面之间有共同的方向关系
- 有些边很清楚，有些边只剩痕迹，有些边化进背景

而不是几片边界完整的硬片。

### 6.2 核心设计

`镜面切片` 要保留主体存在感，但不再靠完整对象轮廓。

新的结构应该由三层组成：

1. 背景流向场  
   背景本身先具备倾斜的速度感、明暗流向和色带。

2. 切面质量块  
   从背景场中提取出 3 到 5 处主切面区域，它们共享同一组方向关系。

3. 边界融化层  
   切面边界只在局部保留亮边或暗边，剩余区域化进背景。

### 6.3 实现顺序

1. 先画背景里的流向场
2. 再生成主切面区域
3. 最后做局部边缘显影和局部边缘融化

### 6.4 参数调整

保留或新增：

- `切面数量`
- `倾角族谱`
- `面宽级差`
- `边界融化`
- `高光显影`

弱化或替换：

- `切片倾角` -> `倾角族谱`
- `厚薄反差` -> `面宽级差`
- `边缘清晰度` -> `边界融化`

## 7. 代码结构建议

### 7.1 长虹玻璃

外部入口仍保留 `reeded-column`，避免影响现有状态结构、预设和随机混搭。

但内部实现改成：

- `buildReededFieldComposition`
- `drawReededFieldBase`
- `drawReededFieldRefraction`
- `drawReededFieldFocus`

也就是说，语义从 `column` 转成 `field`，只是对外 key 不变。

### 7.2 镜面切片

保留 `mirror-slices` 这个入口，但内部实现改成：

- `buildMirrorFieldComposition`
- `drawMirrorSliceMass`
- `drawMirrorSliceEdges`
- `drawMirrorSliceMelt`

重点不是画完整对象，而是画从背景里浮出的切面质量块。

### 7.3 强调层

`drawAccentLayer()` 对这两个主体都只做很轻的收尾，不再承担主要信息。

## 8. 验收标准

### 8.1 长虹玻璃

- 第一眼看到的是整张画面被纵向介质统治
- 不再有完整闭合柱体轮廓
- 局部增强区存在，但不读成单独物体

### 8.2 镜面切片

- 第一眼看到背景里有切面浮现
- 切面边界有断裂和融化，不是每片都完整
- 画面重心来自整组关系，不是某一片硬对象

### 8.3 共通标准

- 两者都更能融入整张壁纸
- 手机屏缩小后主体关系仍能被识别
- 导出、自定义组合、随机混搭和移动端布局不退化

## 9. 结论

这次不应该继续把 `长虹玻璃` 和 `镜面切片` 当作两个“需要画得更像”的对象，而应该分别改成更适合壁纸的结构：

- `长虹玻璃` 是全画面的介质场
- `镜面切片` 是从背景里浮出的半融化切面

只有先把这两者都变成画面本身的结构，而不是贴上去的主体，材质感和抽象完成度才有机会同时成立。
