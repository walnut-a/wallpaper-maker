# 壁纸组合化模块设计文档

## 1. 背景

当前项目已经具备以下能力：

- 单文件、纯前端、静态托管可用
- 打开页面直接出图
- 提供多个预设风格
- 支持实时预览、本地导出和参数调节

现阶段的主要限制是：每个风格预设仍然是整包绑定的。用户可以调参数，但不能把“背景气氛”“主体图形”“纹理覆盖”拆开自由组合。这样会影响后续的可玩性，也会让新风格持续堆在预设列表里，越来越重。

这次改造的目标，是把当前风格体系升级成“模块化组合系统”，同时保留现有预设作为推荐入口。

## 2. 目标与非目标

### 2.1 目标

- 将当前风格系统拆分为 `背景`、`主体`、`纹理` 三个可组合模块
- 保留现有预设，作为推荐组合继续存在
- 支持用户手动选择三类模块并实时组合出新壁纸
- 支持“随机参数”和“随机混搭”两种不同粒度的随机方式
- 保持单文件结构，不引入构建链和外部依赖
- 保持当前预览、导出、尺寸、自定义参数等能力不退化

### 2.2 非目标

- 不做专业级图层系统
- 不做用户自定义保存组合、收藏、命名管理
- 不做无限扩展的节点编辑器式界面
- 不在这次改造里重写全部绘制算法

## 3. 核心设计判断

### 3.1 不是取消预设，而是让预设变成“推荐组合”

首页仍然保留预设卡片。它们不再代表一整套无法拆开的风格模式，而是代表一组经过挑选的模块搭配和默认参数。

用户点预设时，实际上是在应用：

- 背景模块
- 主体模块
- 纹理模块
- 调色板
- 通用参数默认值
- 模块参数默认值

这样既保住“打开就能用”的体验，也给了后续自由组合的空间。

### 3.2 自由组合是第二入口，不抢第一入口

页面依然优先服务“直接出图”。

因此界面顺序不变，只是在预设区域下方新增一组轻量的模块选择控件：

1. 组合预设
2. 自由组合
3. 尺寸与基础参数
4. 快捷动作
5. 高级参数

这样能避免页面一下子变成专业工作台。

## 4. 信息架构

### 4.1 桌面端

左侧控制、右侧预览不变。

控制区顺序改为：

1. 组合预设
2. 自由组合
3. 尺寸与基础参数
4. 快捷动作
5. 高级参数

### 4.2 自由组合区

自由组合区包含三个字段：

- 背景气氛
- 主体图形
- 纹理覆盖

每个字段采用单选下拉，不做过重的卡片列表。

### 4.3 当前状态显示

当用户直接点击预设时，当前状态显示为对应预设名。

当用户手动修改背景、主体或纹理任意一个模块时，当前状态切换为“自定义组合”，但不弹出打断式提示。预览区底部标签同步显示实际模块组合结果。

## 5. 模块系统

### 5.1 背景模块

第一版建议提供：

1. 柔雾渐变
2. 深场冷光
3. 暖冷对冲
4. 玻璃漫反射

职责：

- 负责底色关系
- 负责大面积光感和空间气氛
- 不负责清晰主体轮廓

### 5.2 主体模块

第一版建议提供：

1. 极简大形体
2. 色块软构成
3. 流动带状
4. 长虹玻璃柱面

职责：

- 决定画面的主要构图和视觉重心
- 承担最明显的形状特征
- 不直接带入纹理细节

### 5.3 纹理模块

第一版建议提供：

1. 无
2. 轻网格
3. 蜂窝
4. 细颗粒
5. 玻璃肌理

职责：

- 只负责表面秩序和触感
- 默认透明度克制，不喧宾夺主

## 6. 预设如何映射到模块

现有预设重定义为推荐组合：

1. 极简秩序感  
   柔雾渐变 + 极简大形体 + 轻网格

2. 大色块构成  
   暖冷对冲 + 色块软构成 + 细颗粒

3. 科技纹理感  
   深场冷光 + 极简大形体 + 蜂窝

4. 有机生成感  
   柔雾渐变 + 流动带状 + 细颗粒

5. 长虹玻璃感  
   玻璃漫反射 + 长虹玻璃柱面 + 玻璃肌理

预设除了绑定模块，也绑定默认参数。用户切换预设时，三个模块和对应参数一并切换。

## 7. 状态模型

当前状态从单一 `preset` 扩展为以下结构：

- `preset`
- `backgroundMode`
- `shapeMode`
- `textureMode`
- `palette`
- `density`
- `complexity`
- `seed`
- `advanced`

其中：

- `preset` 用于记录当前是否处于推荐组合
- `backgroundMode`、`shapeMode`、`textureMode` 用于真正驱动渲染
- 当模块被手动修改后，`preset` 进入 `custom` 状态

## 8. 参数体系

### 8.1 通用参数

保持现状，继续放在基础参数区：

- 尺寸预设
- 宽度
- 高度
- 调色板
- 密度
- 复杂度
- 随机种子

### 8.2 组合参数

这部分用于跨模块统一调节，建议放在高级参数顶部：

- 边缘软化
- 光晕半径
- 明暗反差
- 颗粒细度

### 8.3 模块专属参数

根据当前选中的模块动态出现。

例如：

- 选择蜂窝纹理时，出现 `蜂窝尺度`
- 选择流动带状主体时，出现 `漂移幅度`、`流向拉伸`
- 选择长虹玻璃主体时，出现 `棱线密度`、`折射强度`

高级参数区按来源分组显示：

1. 组合控制
2. 背景控制
3. 主体控制
4. 纹理控制

## 9. 随机策略

### 9.1 随机参数

只随机当前组合下可见参数：

- 不改背景模块
- 不改主体模块
- 不改纹理模块
- 只改参数、种子、调色板

适合围绕当前方向继续探索。

### 9.2 随机混搭

随机背景、主体、纹理组合，并生成一套初始参数。

但这里不能完全无约束随机，而要通过兼容规则限制：

- 长虹玻璃柱面优先搭配玻璃漫反射、柔雾渐变
- 蜂窝优先搭配深场冷光、极简大形体
- 玻璃肌理不优先与蜂窝叠加
- 流动带状不优先搭配过强的硬质网格

随机混搭的目标不是“什么都能拼”，而是“多数情况下仍然好看”。

## 10. 渲染结构

渲染顺序调整为：

1. 背景模块绘制
2. 主体模块绘制
3. 强调层绘制
4. 纹理模块绘制

保留现有绘制函数，但逐步改为按模块调用，而不是按预设调用。

第一阶段不要求彻底重写现有函数，只要求完成映射：

- 将现有背景逻辑抽到背景模块
- 将现有主体逻辑抽到主体模块
- 将现有纹理逻辑抽到纹理模块

## 11. 实现策略

### 11.1 第一阶段

- 增加模块配置表
- 扩展状态结构
- 增加自由组合 UI
- 保持现有绘制函数，先完成模块映射
- 预设改为推荐组合

### 11.2 第二阶段

- 优化模块参数分组
- 优化“自定义组合”显示
- 增加随机混搭逻辑
- 调整各模块默认兼容关系

### 11.3 第三阶段

- 对部分算法做进一步拆分
- 提升组合间的视觉一致性
- 减少参数名称和实际效果脱节的问题

## 12. 风险与应对

### 12.1 风险：参数面板变复杂

应对：

- 预设仍然放第一入口
- 自由组合只提供三个单选
- 高级参数继续折叠
- 只显示当前模块相关参数

### 12.2 风险：随机混搭经常翻车

应对：

- 引入兼容矩阵
- 随机结果优先从推荐组合附近做扰动
- 对某些明显冲突的搭配直接降权或禁用

### 12.3 风险：单文件逻辑越来越重

应对：

- 用配置对象整理模块定义
- 保持渲染入口清晰
- 避免把 UI 判断和绘制逻辑继续缠在一起

## 13. 验收标准

- 用户可以继续通过预设一键出图
- 用户可以单独修改背景、主体、纹理三类模块
- 手动修改模块后，状态正确切换为“自定义组合”
- 高级参数只展示当前组合相关参数
- `随机参数` 与 `随机混搭` 行为清晰区分
- 预览、导出、自定义尺寸、移动端布局都保持可用
- 在 GitHub Pages 环境中仍可直接使用

## 14. 最终结论

下一步不应继续只加新的整包预设，而应把现有风格系统升级成“推荐组合 + 自由组合”双轨结构。

这样既能保留当前工具的轻快体验，也能把可玩性明显往前推一大步，同时不需要把项目改造成复杂平台。
