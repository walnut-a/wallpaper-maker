# 材质主体重构设计文档

## 1. 背景

当前项目已经完成主体图形扩展，新增了：

- 流体光团
- 拱门柱影
- 镜面切片

同时保留了：

- 长虹玻璃柱面

从最近一轮实际出图来看，`镜面切片` 和 `长虹玻璃柱面` 都已经有了“像某种材质”的提示，但完成度还不够高。

现在的主要问题不是参数不够多，而是画法顺序不对：

- `镜面切片` 更像几条模糊竖带加一笔高光，切面关系不够明确
- `长虹玻璃柱面` 更像背景上叠了条纹遮罩，柱体本身没有先立住

这次迭代的目标，是把这两个主体都改成“抽象主形体先成立，再用少量物理线索提味”的版本。

## 2. 目标与非目标

### 2.1 目标

- 重构 `镜面切片` 的主体画法，让切面关系先成立
- 重构 `长虹玻璃柱面` 的主体画法，让柱体和折射关系先成立
- 收紧这两个主体的高级参数，只保留真正影响结果的控制
- 保持当前单文件、Canvas 2D、静态托管结构不变
- 不影响现有的导出、尺寸、自定义组合、随机参数和随机混搭

### 2.2 非目标

- 这次不新增新的主体模块
- 不新增新的背景或纹理模块
- 不追求写实物理渲染
- 不改成 WebGL、Shader 或多文件工程

## 3. 现状问题

### 3.1 镜面切片

当前画面的主要问题有三类：

1. 没有清楚的切面层级  
   现在更多是模糊竖向条片，缺少 3 到 5 个真正值钱的主切面。

2. 物理线索太像效果叠加  
   主要依赖模糊、亮条、乘色阴影，像后期特效，不像切面本身的亮暗关系。

3. 抽象构图不够稳  
   就算不把它理解成材质，也还不是一张足够成立的抽象壁纸。

### 3.2 长虹玻璃柱面

当前也有三类问题：

1. 条纹先出现，柱体后出现  
   第一眼看到的是竖向纹理，而不是柱体。

2. 折射关系不够明确  
   条纹更多是在盖一层纹理，没有形成“背景穿过柱体后被轻微压缩和偏移”的感觉。

3. 抽象提炼还不够  
   主体和背景糊在一起，既不够像玻璃，也不够像提炼过的抽象形体。

## 4. 方案比较

### 方案 A：物理优先

把这两个主体尽量往真实材质靠。

优点：

- 材质指向更明确
- 更容易让人第一眼理解“这是什么”

缺点：

- 实现成本高
- 很容易超出当前项目的单文件 Canvas 边界
- 一旦做不够像，反而更假

### 方案 B：抽象优先

把这两个主体都当作抽象图形语言，不强调真实材质。

优点：

- 更稳
- 更容易做出高完成度

缺点：

- 会丢掉你现在已经建立起来的“玻璃 / 切片”方向
- 辨识度不一定比现在更强

### 方案 C：混合路线

先把抽象主形体立住，再用少量物理线索去提示材质。

优点：

- 最适合当前项目
- 既能保住抽象壁纸的完成度，也能保留材质感
- 实现复杂度还在可控范围内

缺点：

- 需要收掉一部分现在已经写进去的效果层
- 默认参数需要重新打磨

### 推荐

采用 `方案 C`。

## 5. 核心设计判断

### 5.1 主体要先像“形体”，再像“材质”

这两个主体后面都应该遵循同一个顺序：

1. 先画主形体
2. 再做光学调制
3. 最后做轻量强调

而不是一开始就把高光、模糊、条纹全叠上去。

### 5.2 物理线索要克制

保留最关键的两三件事就够了：

- 亮边和暗边
- 遮挡关系
- 柱体内部轻微折射

不要追求完整物理拟真。

### 5.3 抽象图必须独立成立

就算用户完全不把它理解成“玻璃”或“切片”，画面也应该本身成立。

## 6. 镜面切片重构方案

### 6.1 目标效果

用户第一眼应该看到：

- 3 到 5 个主要切面
- 统一但有微差的倾角
- 明确的前后遮挡
- 每片切面都有亮边、暗边和主面

而不是几条模糊光带。

### 6.2 绘制顺序

1. 主切面层  
   先确定切片数量、倾角、宽度级差、前后顺序。

2. 边缘层  
   在切面边缘加亮边和暗边，建立“片”的存在感。

3. 局部高光层  
   只在一两片主切面上放小范围高光，不铺满所有切片。

4. 氛围收尾层  
   只保留非常轻的光晕，避免又回到“模糊遮罩”。

### 6.3 参数

建议保留：

- `切片数量`
- `倾角`
- `厚薄反差`
- `边缘锐度`

建议移除或弱化：

- 与 `切片数量` 含义重复的 `主形体数量`

## 7. 长虹玻璃柱面重构方案

### 7.1 目标效果

用户第一眼应该先看到：

- 1 根主柱体，或 2 根错位柱体

第二眼才看到：

- 柱体内部的竖向棱线
- 柱体造成的轻微折射和压缩

### 7.2 绘制顺序

1. 柱体轮廓层  
   先建立柱体轮廓、宽度、位置和重心。

2. 柱体内部调制层  
   只在柱体内部画纵向棱线，不把纹理盖满整张图。

3. 折射层  
   用轻微的明暗和偏移去提示背景穿过柱体后的变化。

4. 高光收尾层  
   只保留一条主高光或一小片局部高光。

### 7.3 参数

建议保留：

- `柱面数量`
- `柱体宽度`
- `棱线密度`
- `折射幅度`

建议改名：

- `折射强度` -> `折射幅度`

## 8. 代码结构建议

当前不需要把 `index.html` 拆文件，但建议把这两个主体各自拆成更清楚的内部步骤。

### 8.1 镜面切片

建议拆成：

- `drawMirrorSliceCore`
- `drawMirrorSliceEdges`
- `drawMirrorSliceHighlights`

### 8.2 长虹玻璃柱面

建议拆成：

- `drawReededColumnCore`
- `drawReededColumnInternalRidges`
- `drawReededColumnRefraction`

### 8.3 强调层

`drawAccentLayer` 不再承担这两个主体的主要信息，只负责最后一点点提亮。

## 9. 验收标准

### 9.1 镜面切片

- 第一眼能数出主要切面
- 至少有一两处清楚的遮挡关系
- 不再像模糊竖条特效

### 9.2 长虹玻璃柱面

- 第一眼先看到柱体，而不是纹理
- 棱线只在柱体内部成立
- 背景穿过柱体后有轻微折射感

### 9.3 共通标准

- 就算不理解材质指向，抽象画面本身也成立
- 手机屏上主形体仍然能被看清
- 导出和移动端预览不退化

## 10. 结论

这次不应该继续靠调参数去救这两个主体，而应该先调整它们的画法顺序。优先把主形体立住，再用少量物理线索去提味，才有机会同时得到：

- 更高的抽象完成度
- 更明确的材质暗示
- 更适合手机桌面的成片率
